## Documentation Maintenance

This documentation is maintained alongside the code. When making changes:

1. Update relevant documentation files
2. Ensure Mermaid diagrams are up-to-date
3. Add examples for new features
4. Update API reference for interface changes

## Diagrams

The documentation uses [Mermaid](https://mermaid.js.org/) for diagrams, which render automatically on GitHub and many Markdown viewers. The Developer Guide includes:

- **Architecture Diagram**: Shows high-level component structure
- **Data Flow Sequence**: Illustrates event processing workflow
- **Classification Decision Tree**: Explains event type determination
- **Reference Matching Process**: Details coordinate matching logic
